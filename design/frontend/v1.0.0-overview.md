# v1.0.0 프런트엔드 오버뷰

## 1. 목적
- v0.x 단계에서 구축한 인증/게임/소셜/토너먼트/관전/리플레이 UI를 v1.0.0 기준으로 정리해 실사용자가 흐름을 따라갈 수 있도록 한다.
- 주요 페이지, 상태관리, 실시간 연동, 성능/UX 안정화 포인트를 요약한다.

## 2. 기술 스택
- React + TypeScript + Vite, 라우팅은 `react-router`.
- 서버 상태: React Query 기반, 실시간: WebSocket 클라이언트 헬퍼(`hooks/useWebSocketClient`).
- 전역 스타일: Tailwind 유사 유틸리티 + 페이지별 CSS, 다크모드 대응 최소화.

## 3. 핵심 화면 및 흐름
- 인증/프로필
  - `RegisterPage` → `LoginPage` → `ProfilePage` 순으로 토큰 저장, 내 정보 조회, 아바타/닉네임 수정 지원.
  - OAuth 로그인 시 백엔드에서 발급받은 토큰을 동일 스토리지에 저장.
- 게임/랭킹
  - `LobbyPage`에서 NORMAL/RANKED 매칭 요청, `GamePage`에서 캔버스 렌더링과 입력 전송.
  - 경기 종료 후 `LeaderboardPage`로 이동해 최신 랭킹/전적 확인.
- 소셜/채팅
  - `FriendsPage`에서 친구 추가/차단, 초대 수락 시 게임/토너먼트 이동.
  - `ChatChannel` 컴포넌트가 채팅 목록/입력/신고 버튼을 제공하며 WebSocket 재연결 백오프 적용.
- 토너먼트/관전/잡
  - `TournamentPage`에서 생성/참여/시작 제어, 매치 상태는 실시간 이벤트로 갱신.
  - `SpectatorPage`에서 진행 중인 방을 선택해 관전, UI는 플레이어 시야와 동일한 캔버스를 읽기 전용으로 노출.
  - `JobsPage`/`ReplayViewerPage`에서 리플레이 렌더링, 잡 진행률/실패 메시지를 배지로 표시.
- 관리자/운영
  - `AdminPage`에서 사용자 검색, 상태 토글(BAN/MUTE), 최근 로그 확인.

## 4. 상태/에러 처리
- API 호출 실패 시 공통 토스트/배너로 한국어 에러 메시지 노출, 401이면 자동 로그아웃 후 로그인 페이지로 리다이렉트.
- WebSocket은 토큰 만료/네트워크 오류 시 즉시 닫고 지수 백오프로 재연결, 3회 실패 후 사용자에게 수동 재시도 안내.
- React Query 캐시 만료 시간을 랭킹/토너먼트는 짧게(실시간성), 프로필/환경 정보는 길게 유지.

## 5. 성능/UX 안정화
- 긴 목록(랭킹, 작업, 채팅)은 서버 페이지네이션 + 가상 리스트(Intersection Observer)로 리플로우를 줄였다.
- 게임/관전 캔버스는 requestAnimationFrame으로 렌더링하고, DOM 접근은 최소화하여 60fps 목표를 유지.
- 한글/이모지 텍스트가 UI 전역에서 깨지지 않도록 `utf8mb4` 응답과 locale 옵션을 고정.

## 6. 남은 제한사항
- 모바일 UI는 최소 대응(반응형 레이아웃)만 제공하며, 터치 입력 최적화/가속 제스처는 미완성이다.
- OAuth UI는 액세스 토큰 입력 기반 데모 흐름이며, 실제 카카오/네이버 동의 창과 에러 핸들링은 축약되어 있다.
- 관리자 페이지는 조회/제재 위주이며, 실시간 알람/대시보드 드릴다운은 제공하지 않는다.
