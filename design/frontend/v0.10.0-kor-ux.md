# v0.10.0 프런트엔드 한국어 UX 정비 및 소셜 로그인 UI

## 1. 목적/범위
- 모든 주요 화면의 안내 문구를 한국어 기준으로 확정하고 혼용된 문구를 제거한다.
- 카카오/네이버 OAuth 토큰 입력 기반 로그인 UI를 제공한다.
- 시간 표시를 Asia/Seoul 기준으로 고정하고 한글 폰트 가독성을 개선한다.

## 2. 외부 동작
- 로그인 페이지
  - 기본 ID/비밀번호 로그인 유지.
  - 소셜 로그인 패널: 제공자 셀렉트(카카오/네이버) + 액세스 토큰 입력 + 로그인 버튼.
  - 성공 시 기존 리다이렉트 경로(기본 `/lobby`)로 이동.
- 시간 표기
  - `toLocaleString('ko-KR', { timeZone: 'Asia/Seoul' })` 적용: 프로필/친구/로비/관리자 화면.
- 시각적 보정
  - 기본 폰트 라인하이트 1.6, `word-break: keep-all`로 한글 단어 단위 줄바꿈 유지.
  - Admin 사용자 행에 로그인 공급자/로케일 배지 표시.

## 3. 내부 구조
- 상태/타입
  - `UserProfile`, `AdminUser` 타입에 `authProvider`, `locale` 추가.
  - `AuthProvider` 컨텍스트에 `loginWithOAuth(provider, accessToken)` 추가.
- API
  - `loginWithProvider(provider, accessToken)` → `/api/auth/oauth/{provider}` 호출.
- UI 반영
  - `LoginPage`: 소셜 패널 추가, 오류 메시지 한국어 통일.
  - `FriendsPage`, `LobbyPage`, `AdminPage`, `ProfilePage`: KST 포맷 헬퍼 적용.

## 4. 테스트/검증
- 수동 확인 포인트
  - 잘못된 액세스 토큰 입력 시 한국어 오류 메시지 노출.
  - 카카오→네이버 전환 시 셀렉트/버튼 라벨이 즉시 변경.
  - 관리자 화면에서 authProvider/locale 배지가 정상 표시.
- 자동 테스트
  - 기존 Jest/React Testing Library 테스트는 동일하게 통과해야 하며, 추가 API 호출 모킹 시 `loginWithOAuth` 경로를 반영한다.

## 5. 남은 과제
- 실제 OAuth 리다이렉트/인가 코드 수집 UI는 추후 버전에서 추가 구현.
- 공통 시간 포맷터 유틸을 hooks/utils로 분리해 재사용성을 높일 여지 있음.
